# PKG-049: Dashboard Loading & Skeleton States

**Status**: 🤖 auto-ready → auto-implemented
**Source**: QA 2026-07-15 — Phase 1 gap-components #4, stores-gap M7
**Priority**: 🔴 Critical (user-visible)
**Scope**: ✅ Non-core — UI components only

## Problem

Noema dashboard shows "No agents loaded" and "No crons loaded" during the initial 15-25s data load, misleading users into thinking the system is broken. There are no loading indicators, skeleton screens, or progressive disclosure — users see empty states instead of loading states.

## Solution

Add a `LoadingSkeleton` shared component and per-tab loading states:

1. **`LoadingSkeleton.svelte`** — Pulse-animated skeleton card component (reusable)
2. **`DashboardLoading.svelte`** — Top-level loading overlay for initial SSR wait
3. **Per-tab loading checks** — Each tab checks if data is ready before rendering content
4. **`ErrorMessage.svelte`** — Reusable error display with retry support

## Implementation

- `src/lib/components/shared/ErrorMessage.svelte` — Error banner with optional retry button
